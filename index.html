花草苑 —— 你的私人植物志

一款面向普通人的数字花草档案工具。你可以一键识别路边那株你感兴趣的花草，填写相遇地点和心情随笔，生成独属于你的植物卡片，并且存入电子花草苑持续关注。

市场破局：去社交化的私人档案

市面识花产品（如形色、花草志）过于商业化或强学术社交，忽视了普通人的情感留存。花草苑反其道而行，定位为纯粹的私人数字博物馆，通过 AI 视觉识别开启“命名即拥有”的情感连接。

多模态AI 情感连接 去社交化

痛点洞察：唤醒城市中的“背景板”

认知断层：每日路过的街边植物，因无法识别而沦为无意义的背景。
记录零散：植物四季的生长变化散落相册，缺乏结构化的“我与植物的专属故事”归档。

目标用户：追求生活质感、希望通过自然观察缓解城市焦虑的个体。

AI 架构与技术决策

以产品思维驱动大模型选型与 Vibe Coding 敏捷交付

模型选型：Qwen3.5-Omni-Flash

多模态原生：保障花卉细微纹理的高精度识别。
商业性价比：高并发下的低调用成本，支撑独立产品的可持续运转。
本土生态：中文语料库储备深厚，在处理中国传统植物寓意与文化意蕴时，准确率与表现力优于海外模型。

开发工具：全栈 AI 交付链路

采用 Gemini + Cursor 的 AI-Native 工作流。
由 Gemini 协助完善产品逻辑与边界约束，通过 Cursor IDE 实操验证。利用自然语言描述需求，极大降低了产品经理将“业务构想”转化为“可运行代码”的工程门槛。

系统提示词设计 (Prompt Engineering)

人设注入：“浪漫的资深植物学家”，兼顾学术严谨与温情。
结构化约束：强制 JSON 输出，确保前端渲染完美闭环。
异常前置：植入“非植物检测”拒绝逻辑，从源头阻断多模态幻觉。

↓

体验演进与 MVP 交付

极简闭环：拍一拍 → 识一识 → 写一写 → 存一存

视觉与交互美学

极简呼吸感：绿色与白色主调，卡片式 UI 营造数字博物馆的高级感。
时间轴记忆：“和它的故事”不仅记录科普，更支持随笔和四季照片留存，大幅拉长产品生命周期与用户粘性。

不确定性治理 (Badcase 优化)

面对易混淆长尾样本（如“牡丹”与“芍药”），通过 Prompt 补充典型差异特征进行范例引导（Few-shot）；增加传统视觉预处理规则，强化对背景杂物的剔除，提升容错率。
